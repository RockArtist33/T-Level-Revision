    # 2.1.4 Understand how to manage variables within a program, including

    In a program, variables are used to store and manage data. Variables can be either local or global, and the choice of which type to use depends on the specific needs of the program.

    Local variables are variables that are declared and used within a specific function or block of code. They are only accessible within the function or block where they are declared, and they are not accessible from outside that function or block. Local variables are useful for storing temporary data that is only needed within the function or block where they are declared, and they can help to avoid conflicts with global variables that have the same name.

    Global variables, on the other hand, are variables that are declared and used outside of any specific function or block of code. They are accessible from anywhere in the program, and they can be used to store data that needs to be shared across multiple functions or blocks. Global variables are useful for storing data that is used throughout the program, but they can also cause conflicts if multiple functions or blocks try to modify the same global variable.

    In general, local variables should be used whenever possible, because they are more specific and have a smaller scope. Global variables should be used only when the data they store needs to be shared across multiple functions or blocks.

    In terms of naming conventions, variables should be given meaningful names that accurately reflect the data they store. This can make the code easier to read and understand, and it can help to avoid confusion.

    In terms of case, there are several conventions that are commonly used for naming variables. Camel case, also known as lower camel case, is a convention where the first word is lowercase and subsequent words are capitalized, with no spaces or punctuation. For example, “myVariable” is an example of a camel case variable name.

    UPPER CASE, also known as all caps, is a convention where all the letters in the variable name are capitalized, with no spaces or punctuation. For example, “MY_VARIABLE” is an example of an UPPER CASE variable name.

    Ultimately, the choice of naming convention is a matter of personal preference and the specific needs of the program. The important thing is to choose a convention and be consistent in its use.
